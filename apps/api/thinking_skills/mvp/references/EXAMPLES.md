# MVP 实验设计示例

## 示例 1：SaaS 产品新功能验证

**背景**：团队想为项目管理工具增加"AI 自动周报"功能，但不确定用户是否愿意付费。

**实验卡**：

| 字段 | 内容 |
|------|------|
| **假设** | 如果提供 AI 自动生成周报功能，30% 的免费用户会升级付费版 |
| **MVP 形态** | 落地页 + 假门测试（点击"开通"按钮记录意向，显示"即将上线"） |
| **核心指标** | 点击"开通"按钮的转化率 |
| **成功阈值** | 转化率 ≥ 8%（行业基准 5%） |
| **时间窗口** | 7 天 |
| **投入成本** | 设计 0.5 天 + 前端 1 天 = 1.5 人天 |
| **成功路径** | 转化率 ≥ 8% → 立项开发，2 周交付 MVP 版本 |
| **失败路径** | 转化率 < 5% → 用户访谈 10 人，重新定义价值主张 |

**结果**：转化率 11%，进入开发阶段。

## 示例 2：线下门店数字化验证

**背景**：连锁奶茶店想推出"小程序预点单"，减少排队时间。

**实验卡**：

| 字段 | 内容 |
|------|------|
| **假设** | 如果提供小程序预点单，午高峰排队时间减少 50%，复购率提升 10% |
| **MVP 形态** | 1 家门店试点，用企业微信群接收预点单（人工处理） |
| **核心指标** | 午高峰平均等待时间 + 试点门店周复购率 |
| **成功阈值** | 等待时间 < 5 分钟（当前 10 分钟）；复购率 +5% |
| **时间窗口** | 2 周 |
| **投入成本** | 1 名店员专职处理群消息，无开发成本 |
| **成功路径** | 达标 → 开发小程序 MVP（4 周），3 家门店扩展 |
| **失败路径** | 未达标 → 分析是需求不足还是执行问题，调整后再测 |

**结果**：等待时间降至 4 分钟，复购率 +7%，启动小程序开发。
