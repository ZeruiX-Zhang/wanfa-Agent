# 5W2H 应用示例

## 示例 1：业务场景 — 客户投诉率上升

| 维度 | 分析 |
|------|------|
| **What** | 月度客户投诉率从 2% 上升至 5% |
| **Why** | 物流合作方更换后配送时效下降 |
| **Who** | 华东地区 B2C 订单客户（占总量 40%） |
| **When** | 2024年3月更换物流商后开始 |
| **Where** | 最后一公里配送环节 |
| **How** | 包裹平均延迟 2 天，且无主动通知 |
| **How much** | 月均多出 1500 单投诉，客服成本增加 ¥4.5 万 |

**结论**：更换物流商后华东地区配送时效恶化，需立即启动备选物流商评估。

## 示例 2：技术场景 — API 响应超时

| 维度 | 分析 |
|------|------|
| **What** | 搜索 API P99 延迟从 200ms 飙升至 2.5s |
| **Why** | 新上线的推荐算法在热门查询时触发全表扫描 |
| **Who** | 所有使用搜索功能的终端用户（日活 12 万） |
| **When** | v2.3 部署后（周三 14:00） |
| **Where** | 推荐服务 → PostgreSQL 查询层 |
| **How** | 推荐模块对未建索引的 `user_preferences` 表做 JOIN |
| **How much** | 影响 12 万日活用户，搜索放弃率 +35%，预估日损失 ¥8 万 |

**结论**：紧急为 `user_preferences.category_id` 添加索引，并对推荐查询增加超时熔断。
