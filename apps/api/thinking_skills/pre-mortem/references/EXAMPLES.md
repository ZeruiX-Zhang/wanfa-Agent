# 预验尸（Pre-Mortem）示例

## 示例 1：新支付系统上线

**项目**：将支付系统从第三方聚合切换为自建支付网关
**假设**：项目已经失败了，我们来分析可能的原因。

| 失败场景 | 严重度 | 概率 | 预防措施 |
|----------|--------|------|----------|
| 资金对账出错，商户少收款 | 🔴 高 | 中 | 上线前用影子模式跑 2 周对账，差异 < 0.01% 才切流 |
| 高峰期网关超时，交易失败 | 🔴 高 | 中 | 压测 3 倍峰值流量；保留旧通道作为降级方案 |
| 合规审查未通过被叫停 | 🟡 中 | 低 | 提前 1 个月提交材料，获得书面预审意见 |
| 用户投诉支付体验变差 | 🟡 中 | 中 | 灰度 5% 用户，监控支付成功率和完成时长 |
| 开发延期导致错过业务窗口 | 🟡 中 | 高 | 砍掉非核心功能，MVP 只做微信+支付宝 |

**最高优先级行动**：
1. 本周启动影子对账模式
2. 下周完成 3 倍压测

## 示例 2：团队远程化转型

**项目**：全公司 200 人从坐班制转为全远程办公
**假设**：半年后转型宣告失败，可能原因是什么？

| 失败场景 | 严重度 | 概率 | 预防措施 |
|----------|--------|------|----------|
| 跨部门协作效率断崖下降 | 🔴 高 | 高 | 建立固定协作时段 + 异步文档规范 |
| 新员工融入困难，3 个月离职率翻倍 | 🔴 高 | 中 | 设计 30-60-90 天远程 onboarding 计划 |
| 信息安全事件（家庭网络泄露） | 🟡 中 | 中 | 强制 VPN + 设备管理 + 安全培训 |
| 管理层不信任，暗中恢复坐班 | 🟡 中 | 中 | 用 OKR 结果衡量产出，月度透明汇报 |
| 员工孤独感导致心理健康问题 | 🟡 中 | 中 | 每周线上社交活动 + 季度线下聚会 |

**最高优先级行动**：
1. 第一周发布异步协作规范文档
2. 第二周上线远程 onboarding 流程
