# 鱼骨图分析示例

## 示例 1：电商订单错发率上升

**问题（鱼头）**：订单错发率从 0.5% 上升至 2.3%

| 类别 | 原因 |
|------|------|
| **人员** | 新员工占比 40%，培训不足；夜班疲劳出错率高 |
| **机器** | 扫码枪老化，识别率下降；WMS 系统偶发卡顿 |
| **材料** | SKU 相似商品包装几乎一致；条码标签模糊 |
| **方法** | 拣货复核流程被跳过（赶单）；无二次校验机制 |
| **环境** | 仓库照明不足；大促期间通道拥挤 |
| **度量** | 错发统计延迟 3 天才反馈；缺少实时告警 |

**主因判定**：⚠️ 方法（复核流程被跳过）+ 材料（相似包装）
**行动**：强制扫码复核才能出库 + 相似 SKU 增加色标区分

## 示例 2：软件系统线上故障频发

**问题（鱼头）**：月均 P1 故障从 1 次增加到 4 次

| 类别 | 原因 |
|------|------|
| **人员** | 核心开发离职，知识断层；Code Review 流于形式 |
| **机器** | 服务器内存不足触发 OOM；监控覆盖率仅 60% |
| **材料** | 第三方 SDK 版本过旧有已知 bug；配置文件无版本管理 |
| **方法** | 缺少灰度发布流程；回滚操作无 SOP |
| **环境** | 测试环境与生产差异大；跨团队接口变更无通知 |
| **度量** | 无 SLO 定义；故障复盘不追踪改进落地 |

**主因判定**：⚠️ 方法（无灰度发布）+ 度量（无 SLO）
**行动**：本周建立灰度发布 SOP + 定义核心服务 SLO 并接入告警
