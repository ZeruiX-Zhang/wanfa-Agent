# PDCA 循环示例

## 示例 1：客服响应时间优化

**目标**：将客服首次响应时间从 5 分钟降至 2 分钟以内

### 第 1 轮 PDCA

| 阶段 | 内容 |
|------|------|
| **Plan** | 分析发现 60% 工单是重复问题；计划引入智能分流 + FAQ 机器人 |
| **Do** | 部署关键词分流规则 20 条；上线 FAQ 机器人覆盖 Top 30 问题 |
| **Check** | 首次响应时间降至 3 分 12 秒；机器人解决率 45%；但复杂问题仍积压 |
| **Act** | 机器人规则纳入 SOP；识别出复杂问题需要专属通道 |

### 第 2 轮 PDCA

| 阶段 | 内容 |
|------|------|
| **Plan** | 为复杂问题建立 L2 专家通道；设置 2 分钟未响应自动升级 |
| **Do** | 配置升级规则；安排 3 名高级客服轮值 L2 |
| **Check** | 首次响应时间降至 1 分 48 秒 ✅；用户满意度 +12% |
| **Act** | 全量推广；下一轮目标：一次解决率 > 85% |

## 示例 2：代码部署频率提升

**目标**：将部署频率从每周 1 次提升至每日部署

### 第 1 轮 PDCA

| 阶段 | 内容 |
|------|------|
| **Plan** | 瓶颈分析：手动测试 4 小时 + 审批 2 小时；计划自动化测试 + 简化审批 |
| **Do** | 编写 CI 自动化测试覆盖核心流程；低风险变更改为 Tech Lead 审批 |
| **Check** | 部署频率提升至每周 3 次；但回滚次数增加（测试覆盖不足） |
| **Act** | 自动化测试纳入 CI 必选项；需补充集成测试 |

### 第 2 轮 PDCA

| 阶段 | 内容 |
|------|------|
| **Plan** | 补充 API 集成测试 + 引入灰度发布（先切 5% 流量） |
| **Do** | 新增 45 个集成测试用例；配置灰度发布管道 |
| **Check** | 部署频率达到每日 1 次 ✅；回滚率从 15% 降至 3% |
| **Act** | 灰度发布成为标准流程；下一轮目标：支持每日多次部署 |
