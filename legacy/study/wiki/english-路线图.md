# 英语（技术文档阅读） - 零基础学习路线图

> 状态：进行中  
> 目标：能流畅阅读英文技术文档、论文、博客

---

## 阶段总览

### Stage 1: 核心词汇与基础语法（3-4 周）
**验收标准**：能读懂简单的 README.md，能识别句子主干

- [ ] 高频 500 词（技术 + 日常混合）
- [ ] 基本句型：SVO, SVC, 被动语态
- [ ] 时态：一般现在时、一般过去时、现在完成时
- [ ] 技术场景常见结构：if...then, in order to, as a result
- [ ] **里程碑**：读完一个 GitHub 项目的 README 并复述大意

### Stage 2: 技术词汇专项（3 周）
**验收标准**：看到技术词汇不需要查词典就能理解

- [ ] 编程高频词：implement, deploy, initialize, iterate, parse...
- [ ] 金融高频词：portfolio, leverage, volatility, hedge, liquidity...
- [ ] 词根词缀法：un-, re-, -tion, -ment, -ize, -able...
- [ ] 缩写与黑话：PR, CI/CD, API, ORM, CRUD...
- [ ] **里程碑**：用英文写一份自己项目的 README

### Stage 3: 长难句攻克（3 周）
**验收标准**：能拆解含 3 个以上从句的复杂句

- [ ] 定语从句（技术文档最常见）
- [ ] 非谓语动词：to do / doing / done 做定语/状语
- [ ] 被动语态的层层嵌套
- [ ] 技术文档常见句式模板：
  - "X is designed to..."
  - "This ensures that..."
  - "Given that..., it follows that..."
- [ ] **里程碑**：精读一篇 Python PEP 文档

### Stage 4: 文档阅读实战（持续）
**验收标准**：能在 30 分钟内读完一篇中等长度的技术文档并提取关键信息

- [ ] Python 官方文档精读
- [ ] Stack Overflow 答案阅读
- [ ] 技术博客文章阅读
- [ ] API 文档阅读（REST API, SDK docs）
- [ ] **里程碑**：纯用英文文档学会一个新库并写出代码

### Stage 5: 论文阅读入门（持续）
**验收标准**：能读懂论文 Abstract + Introduction + Conclusion

- [ ] 论文结构：Abstract, Intro, Method, Results, Discussion
- [ ] 学术英语高频词与句式
- [ ] 如何快速判断一篇论文是否值得细读
- [ ] **里程碑**：独立读完一篇 AI/ML 论文并写出中文摘要

## 英语领域特殊规则

### 验收方式扩展
| 标准 | 含义 | 验证方式 |
|------|------|----------|
| **能读懂** | 不查词典理解文章 80% 以上内容 | 限时阅读 + 复述 |
| **能翻译** | 准确翻译关键段落 | 选段翻译对比 |
| **能仿写** | 能用类似句式写英文 | 英文写作练习 |
| **能听懂** | 能听懂技术 YouTube 视频 | 视频复述 |

### 与其他领域的交叉利用（关键策略！）
英语不需要单独找时间学——**融入编程和金融的学习过程**：

| 编程学习中 | 英语收益 |
|-----------|----------|
| 读 Python 官方文档 | 练技术文档阅读 |
| 看 Stack Overflow | 练问答类文本理解 |
| 写 commit message, README | 练英文写作 |
| 看 YouTube 教程 | 练听力 |

| 金融学习中 | 英语收益 |
|-----------|----------|
| 读 Investopedia | 练金融英语词汇 |
| 读财报 (10-K, 10-Q) | 练商业英语长文 |
| 看 Bloomberg/CNBC | 练财经听力 |

### 每日英语微习惯（不额外占时间）
- 编程学习时：**遇到的生词记到 `raw/english/01-vocabulary/` 中**
- 金融学习时：同上
- 每天结束前：回顾今天积累的 5-10 个新词
