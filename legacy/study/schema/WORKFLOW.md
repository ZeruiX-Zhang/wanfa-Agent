# Schema: 学习工作流 SOP（三领域并行版）

> 三个领域同时推进：软件编程、金融交易、英语（技术文档阅读）  
> 英语不单独占大块时间，融入另外两个领域的学习过程。

---

## 每日安排（建议 90-120 分钟）

### 模式 A：AI 编程 / 大模型日（周一/三/五）

| 阶段 | 时间 | 内容 |
|------|------|------|
| Phase 1 | 10 min | 看 raw 材料：论文/文档/课程笔记（建议英文，一石二鸟） |
| Phase 2 | 25 min | AI 单题推进 → 原理：推导+手写代码 / 工具：实操+review |
| Phase 3 | 10 min | 闭卷：画架构图 + 推公式 + 写核心代码 |
| Phase 4 | 10 min | 更新 wiki + 记录英文生词 |
| 加餐 | 5 min | 读一段论文 / 跑一个实验 |

### 模式 B：金融日（周二/四/六）

| 阶段 | 时间 | 内容 |
|------|------|------|
| Phase 1 | 10 min | 看 raw 材料（建议用 Investopedia 英文版） |
| Phase 2 | 25 min | AI 单题推进 → 案例分析 |
| Phase 3 | 10 min | 闭卷：给案例做决策 + 复述概念 |
| Phase 4 | 10 min | 更新 wiki + 记录英文生词 |
| 加餐 | 5 min | 看盘 / 模拟交易记录（Stage 3 之后） |

### 周日：复盘日

| 阶段 | 时间 | 内容 |
|------|------|------|
| 编程复盘 | 30 min | 本周编程内容输出 |
| 金融复盘 | 30 min | 本周金融内容输出 + 交易日志复盘 |
| 英语汇总 | 15 min | 本周生词汇总 + 精读一篇短文 |
| 规划 | 15 min | 更新"还不会什么"清单 + 下周计划 |

## 单次学习流程（通用）

### Phase 1: 准备（10 min）
1. 把原始材料放进 `raw/[领域]/` 对应文件夹
2. 填写学习验收标准（用 `schema/TEMPLATE-验收标准.md`）
3. 快速浏览材料，标记不懂的地方
4. **如果是英文材料**：标出不认识的词，先猜意思

### Phase 2: AI 单题推进（25 min）
1. 把材料和 `schema/AI-RULES.md` 一起发给 AI
2. 让 AI 先生成**问题地图**
3. 进入一问一答模式：
   - **编程**：概念问答 → 写代码 → debug
   - **金融**：概念问答 → 案例分析 → 决策模拟
   - **英语**：词汇 → 句式 → 段落理解
4. AI 追问失败点，连续卡住 2 次才讲解

### Phase 3: 闭卷检索（10 min）
| 领域 | 闭卷方式 |
|------|----------|
| AI编程/大模型 | 画架构图 + 推关键公式 + 手写核心代码 |
| 金融 | 给新案例做分析决策 + 解释理由 |
| 英语 | 不查词典翻译一段 + 用新学词汇造句 |

通用追加：
- 举例迁移：换一个场景再解释一次
- 反例挑战：让 AI 找反例或边界条件

### Phase 4: 沉淀（10 min）
1. 更新 `wiki/` 中的相关页面
2. 记录错误和新问题到对应的错误与反例页
3. **英文生词**记入 `raw/english/01-vocabulary/`
4. 更新"还不会什么"清单

## 每周流程（90 分钟，周日）

### 1. 汇总（20 min）
- 回顾三个领域的 wiki 更新
- 检查跨领域连接（编程×金融、编程×英语、金融×英语）
- 合并重复内容

### 2. 输出型复盘（50 min）
每个领域选一种输出（本周至少完成 2 个）：
- [ ] 3 分钟讲解稿
- [ ] 一页结构图（Concept Map）
- [ ] 一份 FAQ
- [ ] 一份案例分析
- [ ] 一套测验题（5-10 题）
- [ ] **AI编程专属**：论文笔记 / 核心代码复现 / 架构对比图
- [ ] **金融专属**：一份交易复盘报告
- [ ] **英语专属**：一篇精读笔记或英文短文

### 3. 更新清单（20 min）
- 更新 `wiki/00-学习进度.md`（三个领域分别记录）
- 更新"我还不会什么"清单
- 规划下周学习重点
- 检查领域间的交叉学习机会

## AI 提示词模板

### 开始新主题时（通用）：
```
请扮演学习教练，遵守 AI-RULES.md 中的规则。
当前领域：[编程 / 金融 / 英语]

我的学习验收标准：
1. 问题：___
2. 学完后必须能：___
3. 验收标准：___

材料：[上传]

请先：
1. 给我知识地图
2. 标出前置概念、关键概念、易混概念
3. 从至少 3 个视角提问
4. 先检查我缺哪几块
```

### AI 编程与大模型学习 Prompt：

#### 原理学习时（主线 B）：
```
进入大模型原理教练模式：
- 每个概念给我三层解释：直觉 → 数学公式 → 代码实现
- 数学推导不要跳过，带我一步步推
- 核心代码（attention, backprop, loss）必须让我手写，不准直接给
- 可以用 AI 工具辅助写工程代码（数据处理、可视化），但原理代码手写
- 每学完一个组件，让我画架构图并解释设计动机
- 如果材料是英文论文，同时标注关键术语和重要句式
```

#### 论文阅读时：
```
进入论文阅读教练模式：
- 先让我读 Abstract + Introduction，复述核心贡献
- 然后引导我理解 Method 部分的关键创新
- 帮我区分：哪些是已有工作 vs 本文贡献
- 追问：这个方法的局限是什么？在什么条件下会失效？
- 让我写一份结构化笔记：问题→方法→实验→局限→我的思考
```

#### AI 工具学习时（主线 A）：
```
进入 AI 编程教练模式：
- 我用 AI 工具写的代码，逐行检查我是否真的理解
- 指出 AI 生成代码中的潜在问题（hallucination、安全、性能）
- 帮我建立 review AI 代码的检查清单
- 区分哪些代码可以让 AI 写，哪些必须自己理解
- 如果材料是英文的，标注 3-5 个技术词汇
```

### 金融学习专属 Prompt：
```
进入金融教练模式：
- 每个概念都用真实市场案例来解释
- 每个策略必须先讲"这样做怎么会亏钱"
- 区分清楚：事实 vs 观点 vs 预测
- 不要给我买卖建议，帮我检查分析逻辑
- 闭卷环节给我一个案例，让我做交易决策并解释理由
- 如果材料是英文的，整理出金融术语表
```

### 英语融合学习 Prompt：
```
我正在用英文材料学 [编程/金融]。请同时帮我：
1. 标出重要的英文词汇（给出技术语境下的含义）
2. 挑出 1-2 个值得分析的长难句，拆解结构
3. 主体学习内容正常推进（用中英混合）
4. 学完后让我用英文简短复述一遍核心概念
```

### 闭卷检索（通用）：
```
进入闭卷检索模式。领域：[编程/金融/英语]
我会不看材料给你复述/写代码/做分析。请你：
1. 指出遗漏的关键点
2. 指出理解错误的地方
3. 给一个新场景让我再做一次
4. 找反例或边界条件挑战我
5. 评估理解深度：表层/中层/深层
```
