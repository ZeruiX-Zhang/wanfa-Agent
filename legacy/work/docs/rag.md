# Knowledge RAG

RAG supports Markdown, text, CSV, HTML, and best-effort PDF parsing. Ingestion stores chunk metadata for traceability: document ID, filename, page, section, chunk index, token count, and source path.

Retrieval uses deterministic local embeddings plus BM25 keyword search. Hybrid mode fuses vector and keyword results, then applies a simple rerank fallback when no external reranker is configured.

Answers are grounded in retrieved context. Citations are generated by code from chunk metadata, not by the model.

The desktop Knowledge QA page also runs a local QA orchestration layer inside the RAG Workbench service path. Each question now records `qa_plan`, `evidence_report`, `verification`, `answer_type`, and confidence in the RAG trace, so problem decomposition and evidence gating are visible without leaving the desktop RAG system.

Useful commands:

```bash
python scripts/ingest_docs.py --path data/sample_docs --replace
python scripts/demo_rag.py
```
